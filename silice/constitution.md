## Philosophy

**Paradigma:** Estrategia Cinética 2D / Coevolución — "1 Jugador x Meditando x 1 Jugador"

This is NOT a zero-sum game. There are no losers. The goal is emergent complexity, kinetic meditation, and mutual reflection.

- **Atomic Monolith (Fase 0-1):** No micro-services, no server dependencies. Everything runs in the browser.
- **Progressive Enhancement:** Tauri/Rust backend is Phase 2. Phase 0-1 must be fully self-contained.
- **Pure Functions:** Engine modules must not produce side effects outside their domain. Board.js does not touch the DOM.
- **Config-First:** Every game parameter must be overridable through the configuration object. No magic numbers in engine code.
- **Graceful Degradation:** Optional features (Ollama, Tauri) must fail silently. The game is fully playable without them.
- **Zero Token Waste:** Agents read only the module(s) affected by a task. Never dump the entire codebase.

## Agent Operational Limits

1. **Never edit `codebase.json`** — Auto-generated by Git hooks (Phase 2). During Phase 0-1, updated manually after major structural changes.
2. **Never introduce new dependencies** without updating `constitution.md` Tech Stack table.
3. **Never add global mutable state** — Game state lives exclusively in `TurnManager` and is passed explicitly.
4. **Backlog first:** Every structural change requires a `backlog.json` task entry before implementation.
5. **Module boundaries are sacred:** `engine/` modules are pure logic. `ui/` modules are pure presentation. `db/` handles persistence only.
6. **Canvas rendering lives in `engine/Renderer.js`** — No `ctx.draw*` calls outside this module.
